# Pushing MindMate to GitHub

The project is already a local Git repository with one commit. Follow these
steps to publish it and get the shareable link the Qollabb submission needs.

## 1. Create an empty repository on GitHub
- Go to <https://github.com/new>.
- Name it e.g. `mindmate-mental-health-assistant`.
- **Do not** add a README, .gitignore or licence (the project already has them).
- Click **Create repository**.

## 2. Connect and push
Copy the repository URL GitHub shows you, then run from the project folder:

```bash
git remote add origin https://github.com/<your-username>/mindmate-mental-health-assistant.git
git push -u origin main
```

If you use SSH instead of HTTPS:

```bash
git remote add origin git@github.com:<your-username>/mindmate-mental-health-assistant.git
git push -u origin main
```

## 3. (Optional) Deploy a live demo
The app runs on **Streamlit Community Cloud** for free:
- Go to <https://share.streamlit.io>, sign in with GitHub.
- Click **New app**, pick your repo, branch `main`, main file `app.py`.
- Deploy. The first run installs `requirements.txt` and builds the app.
- (Optional) add `OPENAI_API_KEY` under the app's **Secrets** to enable live AI replies.

> The deployed app will train its models on first run only if you add a small
> startup step, or you can commit the trained `.joblib` artifacts. By default
> the models are git-ignored; to ship them, comment out the
> `models/artifacts/*.joblib` line in `.gitignore`, run
> `python -m scripts.train_models`, then `git add -f models/artifacts` and commit.

## 4. Paste the link in Qollabb
Use either link in the **Final Report Submission → "Link to your work"** field:
- Source code: `https://github.com/<your-username>/mindmate-mental-health-assistant`
- Live demo (if deployed): `https://<your-app>.streamlit.app`

## Before you push, fill in your details
Search-and-replace these placeholders so your name appears everywhere:
- `[Student Name]` and `[Enrolment Number]` in:
  - `docs/report/` chapter front matter is generated by the builder, edit them in
    `scripts/build_report.py` (functions `cover_page`, `certificate_page`,
    `declaration_page`, `acknowledgement_page`) **or** simply edit the final PDF's
    cover/declaration in any PDF editor.
  - `scripts/build_slides.py` / `build_slides_pdf.py` title slide.
  - `LICENSE` (copyright line).
After editing the builders, regenerate with:
```bash
python -m scripts.build_report
python -m scripts.build_slides && python -m scripts.build_slides_pdf
```
